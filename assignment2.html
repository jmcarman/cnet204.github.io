Assignment 2 - Fall 2023 - Due Friday, December 15th at 11:59pm

Overview

In this assignment you are going to learn how to automate deploying a static website from GitHub using GitHub Pages and configure GitHub Pages to use a custom domain. This will allow you to deploy a website directly from your GitHub repository without needing to configure a server. You will not be using AWS in this assignment. Steps for configuring your repository to use GitHub pages are are provided below. You will then use this infrastructure to build a website. This website will be automatically deployed from your GitHub repository. The website specifications are included below.

Access your GitHub repository through a web browser

Login to GitHub and access your repository. Click on Settings (screenshot below):

Click on the Source drop down under Build and Deploy and select Deploy from a branch.

Click on the Branch drop down under Build and Deploy and select Main.

Scroll to the top of your GitHub Pages configuration page and you should see a URL to access your site.

Configuring GitHub Pages with a custom domain

Now you are going to configure GitHub Pages to use a custom domain. You are going to use a sub domain of the domain you registered in assignment 1. Click on Settings in your assignment 2 GitHub repo.

Scroll down to the Custom domain section and enter your sub domain name in the text box. Click Save.

Login to your DNS provider (that you registered in Assignment 1). Add a CNAME record that points to youruser.github.io.

Go back to your GitHub Pages settings. Under Custom Domain, click Check again until you see the message DNS check successful. Then check the box beside Enforce HTTPS.

Creating a Website

Individually or (optionally) in a group of 2 (2 people would submit 1 website): You will complete a multi-page website using the tools and techniques we've covered in Labs 1-10. Choose one of the following topics and fulfill the required pages detailed below. All materials need to be submitted to your GitHub repository and all screenshots uploaded to the dropbox on e.centennial by the due date in order to recieve marks. If you're completing this as a group, only one group member needs to submit the project (and include both names).

Pick a topic from the list below (note the attached resources).

  1. Human Trafficking in Ontario
  2. Affordable Housing and Ending Homelessness
  3. Supporting Aboriginal people
  4. Helping women flee domestic violence
  5. Endangered Species in Ontario
  6. Climate Change initiatives in Ontario
  7. Any of the Poverty issues listed here
  8. Any of the labour related issues listed here

Your site needs to include the following pages:

  1. Index Page (index.html): Providing introductory content relating to your topic
  2. Issue details: go into further detailed information about the issue by describing any of the following:
    • The community
    • The issue
    • Some factors that contribute to the issue
  3. Reflection Practice: Your own personal reflections
  4. The power of social media and websites to promote awareness of the issue
  5. A video describing your issue
  6. A second video discussing how technology has been used to address the inequality
  7. Resource Page: Listing hyperlinks to all websites you have consulted in gathering information for your project

Your website should contain the following features:

  1. Consistent for all pages:
    • Navigation
    • Fonts and Colours
    • Size and Layout
  2. Each page should have
    • a unique title
    • a comment indicating your name(s) and student number. If you're working in a group, every page should contain both members name and student number.
  3. One linked style sheet to implement your CSS in order to ensure uniformity across all pages

Your website will be evaluated for Design, Content, stable transition between pages (navigation options don't move around), relevance (relevant information about your topic on all pages), accessibility, consistency of fonts and colours. Each page will be marked out of 5, with the individual breakdown as follows:

  • Navigation options are consistent, and active class is properly utilized (2 marks)
  • Page layout is consistent across your website (1 mark)
  • Page colours are consistent across your website (1 mark)
  • Content is relevant & each page contains a comment with your name(s) and student number(s) (1 mark)
